## Single most-likely failure path (adversary verdict, accepted)

**Closed-loop fabricated provenance**: an attacker mints `src:`, `span:`, `claim:` records that all reference each other. `quote_hash` is sha256 of attacker-authored text. Validator passes (all FKs resolve internally). No external fetch, no signature, no reviewer independence check. `PROVEN_RELIABLE` laundered.

**Why this is the #1 attack**: it's the only attack that works against the v0.3 design as-spec'd. Every other attack vector requires either crypto break (post-quantum), out-of-band collusion (review-mesh), or implementation bug (NFC drift). This attack works WITHIN the v0.3 spec by following the rules literally.

**v0.4 mitigation** (mandatory before publication):
1. `PROVEN_RELIABLE` claims MUST have basis of ≥2 distinct `source_id`s.
2. At least one basis source MUST satisfy the external-anchor rule (`url` with `location_hash`, OR `git-ref` with immutable commit-sha ref, OR `in-packet` reference committed to `assets_merkle_root`).
3. Validator MUST enforce both checks mechanically.
4. Conformance corpus MUST ship an `attack-closed-loop-provenance/` invalid example that v0.4 validators reject.

## Highest-leverage strategic change (strategist verdict, accepted)

**Ship v0.4 as a JSON-LD profile aligned with RO-Crate, not as a parallel format.** Every AEP packet is also a valid RO-Crate AND a valid JSON-LD document, with `aep:reliability` and `aep:scope` as namespaced predicates. Adoption friction drops from "new format" to "opinionated profile of an existing standard."

Operational consequence: `aep:0.4/jsonld` is the recommended profile. `contexts/aep.context.jsonld` is required for the JSON-LD profile and maps AEP records onto schema.org + PROV-O concepts where stable.

## Adobe `.aep` extension clash (scout finding, mitigated)

The `.aep` extension is already used by Adobe After Effects Project files. The `.aepkg` directory extension is reasonably distinct, but the abbreviation "AEP" is overloaded (After Effects Project; Application Enablement Platform; Avaya).

**Mitigation in v0.4**:
1. Brand "Agent Evidence Packet" explicitly in every README first paragraph; never use the bare abbreviation.
2. Reserve the URI namespace `aep:` (e.g., `aep:Claim`, `aep:Source`) so concept references are unambiguous even when the abbreviation isn't.
3. Keep `.aepkg` as the directory form (no clash) and `.aepkg.zip` as the portable form.
4. If post-publication discovery shows ongoing confusion, v0.5 can introduce an aliasing extension `.evpkg` (evidence-packet) while keeping `.aepkg` as primary.

## Mandatory v0.4 amendments (consolidated list)

These MUST land in v0.4 before publication:

1. **Bump `aep_version` to `0.4`** and update manifest schema.
2. **State-hash algorithm**: explicit NFC normalization + reject BOM + reject CRLF + reject empty-line ambiguity.
3. **Manifest + assets in integrity envelope**: `integrity.manifest_hash` + `integrity.assets_merkle_root` alongside `integrity.state_hash`.
4. **WriteEvent chain integrity**: `pre_state_hash` MUST chain to previous event's `post_state_hash`; first event's `pre_state_hash = sha256("")`.
5. **PROVEN_RELIABLE external-anchor rule**: ≥2 distinct sources + at least one external anchor.
6. **Axiom 8 mechanical enforcement**: same-source-basis collapse → validator rejects PROVEN_RELIABLE.
7. **`source.location` discriminator schema**: `kind` enum (filesystem-path, url, git-ref, in-packet, etc.) with per-kind required fields.
8. **`span.selector` discriminator schema**: `kind` enum with per-kind required fields.
9. **`span.quote_hash` algorithm normative**: NFC + entity-decode + whitespace-collapse, then sha256.
10. **`created_at` format normative**: RFC3339 with UTC `Z` suffix.
11. **`source_type` enum extended**: +5 values (llm_output, tool_output, external_research, human_testimony, derivation).
12. **`provenance_strength` adds `independent_convergent`**: operationalizes axiom 8 in the source record itself.
13. **`inference_label` adds 2 values**: `analogical_transfer`, `cross_packet_synthesis`.
14. **`validation.result` → `schema_result`**: prevents conflation with reliability promotion (LAW-06).
15. **`claim.reliability` adds `GOVERNANCE_RULE`**: DivOmni §02 Amendment A15.
16. **Optional `axis_b_action` field**: action-disposition layer (V11 charter §2.1 two-axis schema).
17. **Threat model rewrite**: 8 sub-sections replacing 5 prose truisms.
18. **JSON-LD profile**: `contexts/aep.context.jsonld` required for `aep:0.4/jsonld`.
19. **Validator MUST fail-closed on**: chain break, hash mismatch, BOM, CRLF, empty PROVEN_RELIABLE basis, same-source basis collapse, missing UNKNOWN reasoning.
20. **Conformance corpus**: 11 fixture packets (valid + invalid families).
21. **Migration tool** (`migrate_v0_3_to_v0_4.py`): detect non-conformant v0.3 packets and emit migration plan.
